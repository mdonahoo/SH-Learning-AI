## Documentation Standards

All documentation follows these principles:

1. **Clarity** - Write for clarity, not brevity
2. **Examples** - Include working code examples
3. **Up-to-date** - Document as you code
4. **Searchable** - Use clear headings and keywords
5. **Accessible** - Assume minimal prior knowledge

## Contributing to Documentation

Found an error or want to improve docs?

1. Documentation files are in Markdown (`.md`)
2. Follow existing format and style
3. Include examples where helpful
4. Update this index if adding new docs
5. Submit PR with clear description
